# Backlog фич

Этот список хранит продуктовые идеи, которые не входят в текущий проход реализации, но должны остаться в поле зрения для следующих версий калькулятора.

| ID | Фича | Идея | Как реализовать | Приоритет |
|---|---|---|---|---|
| F-001 | Импорт SKU из стандартных выгрузок WB/Ozon | Не заставлять клиента заполнять наш шаблон вручную. Пользователь загружает стандартную XLSX/CSV-выгрузку из кабинета Wildberries или Ozon, а калькулятор подтягивает SKU, категории, цену, вес и габариты. | Сделать экран импорта с выбором источника `Wildberries` / `Ozon`, парсингом файла в браузере, маппингом колонок, предпросмотром, валидацией и подтверждением добавления SKU в расчет. Для первой реализации нужны реальные примеры выгрузок WB и Ozon на 3-5 товаров. | Высокий |
| F-002 | Детализация наценки складских операций | Сейчас складские операции PIM.Seller имеют одну общую процентную наценку. Возможно, позже понадобится разнести наценки по приемке, хранению, комплектации и маркировке. | Добавить отдельные поля в админ-панели и расчетный движок по каждой складской операции, сохранив общую таблицу маржи по центру прибыли `Склад`. | Средний |
| F-003 | Пересмотр FBO в марже и первой миле | Закрыто для текущей версии: FBO нужен как ориентир для сравнения с FBS/DBS; первая миля в FBO показывается справочно и не входит в итог, админская маржа FBO скрыта. | Если позже PIM.Seller начнёт продавать услугу первой мили для FBO, вернуть её в итог и маржу отдельным бизнес-решением. | Закрыто |
| F-004 | Формат сценария демо для менеджера | Сейчас сценарий демо зафиксирован как Markdown-шпаргалка `docs/demo-manager-script.md`. Нужно позже выбрать, как менеджеру удобнее пользоваться им в работе. | Рассмотреть варианты: кнопка/панель `Сценарий демо` внутри калькулятора, отдельная защищённая страница `/demo-script`, интерактивный чеклист внутри приложения. Решение принять после первых реальных показов клиентам. | Низкий |
