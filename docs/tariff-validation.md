# Аудит тарифов и формул калькулятора

Документ нужен для сверки бизнес-логики до расширения калькулятора. Формат работы: проходим блоки сверху вниз, подтверждаем источник и формулу, затем закрепляем подтверждённые места golden-тестами. Перед переходом к следующему блоку явно спрашиваем, можно ли считать предыдущий блок закрытым, если это не было сказано пользователем напрямую.

## 1. Метрики SKU

| Блок | Источник | Текущая формула | Статус | Что проверить |
|---|---|---|---|---|
| Литры | `Кейс для Мебели .xlsx` | `длина * ширина * высота / 1000` | Почти подтверждено | В Excel порядок колонок `длина/высота/ширина`, в коде порядок не важен из-за умножения. |
| Объёмный вес | `Кейс для Мебели .xlsx` | `длина * ширина * высота / 5000` | Почти подтверждено | Подтвердить, что делитель `5000` применяется для всех маркетплейсов и схем. |
| Расчётный вес | MVP-допущение | `max(фактический вес, объёмный вес)` | Нужно подтвердить | В Excel часть формул использует литры, часть фактический/объёмный вес. Нужна единая логика для последней мили. |
| Габаритная категория WB/Ozon | Правила маркетплейсов, решение MVP от 2026-06-09, уточнение Ozon от 2026-06-11 | WB: `МГТ`, `КГТ+`, `СГТ`; Ozon: `Стандарт`, `КГТ`. Категория считается автоматически из веса и трёх сторон упаковки. | Подтверждено для текущей версии | В матрице сравнения рядом с названием SKU показываем бейджи только если товар выходит за обычный класс: WB не `МГТ` или Ozon не `Стандарт`. Для WB `КГТ+`/`СГТ` показываем неблокирующие предупреждения. Для WB FBS `СГТ` скидка быстрой сдачи не применяется, а логистика FBS берется из СГТ-строки WB при наличии. Для Ozon КГТ доставка FBO/FBS считается по той же таблице `logistika-fbo-fbs-01052026_1777018200.xlsx` по направлению и ВГХ; отдельное предупреждение не показываем. Для Ozon DBS/RFBS используется RFBS-комиссия и PIM.Seller-доставка. |

## 2. Комиссии маркетплейсов

| Блок | Источник | Текущая формула | Статус | Что проверить |
|---|---|---|---|---|
| WB комиссии | `сomission.xlsx` | Поиск только по `subject`; `category` используется для подбора/фильтрации предмета. Если `subject` пустой или ставка не найдена, схема неполная | Подтверждено технически | Заголовки файла: `Склад WB, %` -> FBO, `Склад продавца - везу на склад WB, %` -> FBS, `Склад продавца - везу самостоятельно до клиента, %` -> DBS. Колонка `экспресс` и `самовывоз` в MVP игнорируются. WB API `/api/v1/tariffs/commission` не используем для комиссий: поле `kgvpPickup` не соответствует FBO-колонке Excel и не будет маппиться на FBO без отдельного подтверждения WB. |
| Ozon FBO/FBS | `Таблица_категорий...xlsx` | Поиск только по `productType`; `category` используется для подбора/фильтрации типа товара, ставка выбирается по ценовому диапазону | Подтверждено технически | Заголовки файла: FBO = `C:H`, FBO Fresh = `I:N` игнорируется, FBS = `O:T`. Диапазоны закреплены тестом: `<=100`, `<=300`, `<=1500`, `<=5000`, `<=10000`, `>10000`. |
| Ozon DBS | `Таблица_категорий...xlsx` | Используются колонки RFBS как DBS | Подтверждено бизнесом | Решение: для Ozon в калькуляторе RFBS считается тем же сценарием, что DBS. Технически RFBS находится в колонках `U:X`. |
| Быстрая сдача | Бизнес-решение от 2026-06-06, новость WB от 2026-06-15 с действием с 2026-06-18 | Применяется только к FBS. WB: для `МГТ` и `КГТ+` ставка комиссии уменьшается на `1,5 п.п.` при сдаче до 13 часов; для `СГТ` расчеты не меняются. Период 13-18 часов у WB считается базовой комиссией без скидки. Ozon: ставка комиссии уменьшается на `3 п.п.` для `СЦ/курьер до 12 часов`, на `2 п.п.` для `СЦ/курьер 12-24 часа` и `ПВЗ/ППЗ до рекомендованного слота`. | Подтверждено бизнесом для текущей версии, согласовано 2026-06-15 | Скидка показывается внутри строки комиссии как эффективная ставка и расчет в скобках: WB `28.5% (30%-1.5%)`, Ozon `45% (48%-3%)`; отдельной строкой скидку не выводим. FBO/DBS не меняются. Текущая галка `Быстрая сдача` моделирует сценарий `до 13 часов` для WB; отдельный выбор окна 13-18 часов в MVP не добавляем. |
| Fallback комиссий | Нет | Если WB/Ozon категория или тип товара не найдены, схема помечается как неполная и показывает предупреждение `Тариф не найден` | Подтверждено бизнесом | Тихие fallback-ставки убраны, чтобы калькулятор не показывал уверенную неверную цифру. |

Тестовое покрытие по этому разделу:

- `wildberries commission columns are mapped to FBO/FBS/DBS schemes`
- `ozon commission columns and price bands are mapped to scheme rates with RFBS as DBS`
- `fast handover discounts apply only to FBS with marketplace-specific rules`
- `missing marketplace commission creates an incomplete scheme instead of a silent fallback`
- `Ozon KGT warnings describe current scheme-specific limitations`

## 9. Схемы и состав статей

| Схема | Текущий состав | Что проверить |
|---|---|---|
| FBO | Первая миля + комиссия маркетплейса + логистика/хранение маркетплейса | В ТЗ первая миля делает селлер или маркетплейс; в коде она всегда включена. Нужно решить, показывать ли её как опциональную. |
| FBS | Первая миля + комиссия маркетплейса + логистика маркетплейса + склад PIM + средняя миля PIM | Подтвердить, что последняя миля в FBS всегда маркетплейс. |
| DBS | Первая миля + комиссия маркетплейса + склад PIM + последняя миля PIM | Подтвердить, что средняя миля и маркетплейс-логистика не нужны. |
