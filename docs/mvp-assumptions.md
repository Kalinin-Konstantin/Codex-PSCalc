# MVP-допущения

Этот список собирает бизнес- и продуктовые упрощения, которые сознательно принимаем для MVP. К нему возвращаемся перед переходом от прототипа к более точной версии калькулятора.

| ID | Блок | Допущение | Почему так для MVP | Где влияет | Что пересмотреть позже |
|---|---|---|---|---|---|
| A-001 | Складские операции PIM.Seller: хранение | Для всех SKU используем тариф хранения EUR-паллеты высотой до 1,8 м: `35,82 руб. * storageDays / itemsPerPallet`. | Не добавляем в интерфейс отдельный выбор высоты паллеты и не усложняем ручной ввод SKU. | FBS и DBS, статья `Хранение PIM.Seller`. | Нужен ли выбор высоты паллеты `1,28 / 1,5 / 1,8 / 2,2 / 2,8 м` для каждого SKU или автоматический подбор по габаритам. |
| A-002 | Складские операции PIM.Seller: комплектация | Тариф комплектации выбираем по фактическому весу SKU, а не по расчётному весу. | Тариф складских операций в DOCX задан как весовой диапазон для обработки товара; для MVP не применяем логистический объёмный вес к складской комплектации. | FBS и DBS, статья `Складские операции PIM.Seller`. | Нужно ли для крупногабаритных товаров вводить отдельное правило складской обработки или оставить физический вес как постоянное правило. |
| A-003 | НДС | Введённую цену SKU считаем ценой с НДС. Тарифы маркетплейсов считаем исходно с НДС и очищаем в режиме `без НДС`; тарифы PIM.Seller считаем исходно без НДС и добавляем НДС в режиме `с НДС`. Основная ставка для MVP: `22%`. | Нужно показывать оба базиса расчёта и не смешивать marketplace gross-тарифы с PIM net-тарифами. | Все схемы, итоговая сумма, `% от цены`, детализация статей. | Подтвердить налоговый режим PIM.Seller и клиента, обработать ставки `без НДС / 5% / 7% / 10% / 22%` при необходимости. |
| A-004 | Быстрая сдача | Галка `Быстрая сдача` влияет только на схему FBS. Для WB FBS с 2026-06-18 применяем `-1,5 п.п. к комиссии` при сдаче до 13 часов для `МГТ` и `КГТ+`; для `СГТ` скидка не применяется. Период WB 13-18 часов означает базовую комиссию без скидки, но отдельный выбор этого окна в MVP не добавляем. Для Ozon FBS применяем скидку как уменьшение ставки комиссии на процентные пункты: `-3 п.п.` для `СЦ/курьер до 12 часов`, `-2 п.п.` для `СЦ/курьер 12-24 часа`, `-2 п.п.` для `ПВЗ/ППЗ до рекомендованного слота`. В подписи комиссии показываем эффективную ставку и расчет, например WB `28.5% (30%-1.5%)`, Ozon `45% (48%-3%)`. | Нужно быстро проверить влияние скидки на экономику без ввода фактического времени сдачи по каждому заказу. | Статья `Комиссия маркетплейса` только для WB/Ozon FBS. | Если позже понадобится точное моделирование SLA, добавить ввод/выбор времени сдачи WB для окон `до 13 часов`, `13-18 часов` и возможных правил просрочки. |
| A-005 | Коммерческая наценка PIM.Seller | Тарифы первой мили, средней мили, последней мили и складских операций PIM.Seller считаем исходной себестоимостью. Коммерческая цена услуги = `себестоимость * (1 + наценка / 100)`, затем применяем выбранный режим НДС. Проценты наценки и маржа видны только в скрытой админ-панели, которая открывается маленькой кнопкой в углу. Админская маржа считается в том же режиме НДС, что и основной расчёт. | Калькулятор используется в переговорах с клиентом, поэтому маржинальность нужно регулировать, но не раскрывать клиенту. | PIM-статьи: `firstMile`, `middleMile`, `lastMile`, `pimReceiving`, `pimStorage`, `pimFulfillment`, `pimLabeling`; админ-таблица маржи по центрам прибыли на 1 шт. | Добавить профили наценок, PIN/скрытый доступ к админ-панели и отдельный отчёт по марже. |
